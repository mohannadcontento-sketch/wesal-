# وصال (Wesal) — دليل تفعيل نظام التوثيق (Email OTP + Password + Phone)

## 🎯 ملخص سريع

النظام بيتعامل مع Supabase Auth Email OTP:
- **التسجيل**: اسم مستعار + رقم موبايل (إجباري) + إيميل + كلمة مرور
- **التأكيد**: كود 6 أرقام بيترسل على الإيميل
- **الدخول**: إيميل + كلمة مرور
- **الحظر**: لو المستخدم اتحظر، مش هيقدر يسجل تاني بالموبايل أو الإيميل

---

## 📋 الخطوات (مقدرش أكثر من 10 دقائق)

### الخطوة 1: إعداد Email Auth في Supabase

1. روح على **Supabase Dashboard** → اختار المشروع بتاعك
2. من القائمة الجانبية، اضغط على **Authentication**
3. اضغط على **Providers**
4. لقي **Email** → تأكد إنها **مفعّلة** (Enable)
5. تحت **Email Auth**, فعّل:
   - ✅ **Confirm email** (مهم — ده بيعمل الـ OTP)
   - ✅ **Secure email change**

> ⚠️ "Confirm email" هو اللي بيعمل كود التأكيد. لو مش مفعّل، المستخدم هيكون مفعل تلقائياً بدون OTP.

---

### الخطوة 2: تشغيل SQL Schema

1. من Supabase Dashboard → روح **SQL Editor**
2. اضغط **"+ New query"**
3. انسخ كل محتوى ملف **`supabase/schema.sql`** والصقه
4. اضغط **Run** (أو Ctrl+Enter)
5. لو ظهرت رسالة **"Success"** → تمام ✅

**الكود ده بيعمل:**
- Trigger تلقائي لإنشاء profile عند التسجيل
- `users.id = auth.users.id` (عشان RLS يشتغل)
- تشفير رقم الموبايل بـ SHA-256 (للحماية)
- حفظ رقم الموبايل الفعلي (للتحقق من الحظر)
- عمود `is_banned` لحظر المستخدمين
- كل RLS policies شغالة

---

### الخطوة 3: التحقق من Environment Variables

في `.env.local` (أو Vercel), تأكد إن عندك:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
GEMINI_API_KEY=your-gemini-api-key-here  # اختياري — للتحليل الذكي
```

---

### الخطوة 4: تخصيص Email Template (اختياري)

لو عايز تعدّل شكل الإيميل اللي بيوصل:

1. Supabase Dashboard → **Authentication** → **Email Templates**
2. اختار **"Confirm signup"**
3. عدّل الإيميل زي ما تحب
4. اضغط **Save**

---

### الخطوة 5: تجربة النظام

1. افتح الموقع
2. اضغط **"ابدأ رحلتك"** أو **"تسجيل الدخول"**
3. في تبويب **"حساب جديد"**:
   - اختار اسم مستعار (3 حروف على الأقل)
   - حط رقم الموبايل (مثلاً: `01012345678`)
   - حط البريد الإلكتروني
   - كلمة مرور (8 حروف على الأقل)
   - وافق على الشروط
   - اضغط **"إنشاء حساب جديد"**
4. **هتلاقي كود تأكيد 6 أرقام على الإيميل**
5. دخل الأرقام الـ 6
6. اضغط **"تأكيد البريد الإلكتروني"** → هتدخل للمجتمع تلقائيًا!

## 🔧 مفاهيم فنية مهمة

### كيف بيشتغل النظام؟

```
التسجيل:
المستخدم                       Supabase Auth
   |                                |
   |── 1. أدخل البيانات ──────────>|
   |   (اسم + موبايل + إيميل + باسورد)
   |                                |── 2. أنشئ auth.users
   |                                |── 3. Trigger → إنشاء profile
   |                                |    في جدول users (مع phone + phone_hash)
   |                                |── 4. أرسل كود تأكيد على الإيميل
   |<── 5. "تم إرسال الكود" ──────|
   |                                |
   |── 6. أدخل الكود ─────────────>|
   |                                |── 7. تحقق من الكود
   |                                |── 8. تفعيل الحساب
   |<── 9. Session + Token ────────|

الدخول:
   |── 1. إيميل + باسورد ────────>|
   |                                |── 2. تحقق + فحص الحظر
   |<── 3. Session ────────────────|
```

### نظام الحظر:

```
المستخدم المحظور                      النظام
   |                                    |
   |── 1. حاول يسجل حساب جديد ───────>|
   |   (موبايل محظور)                   |── 2. فحص الموبايل في جدول users
   |<── 3. "حسابك محظور" ─────────────|
   |                                    |
   |── 4. حاول يسجل دخول ─────────────>|
   |   (حساب محظور)                     |── 5. فحص is_banned
   |                                    |── 6. sign out تلقائي
   |<── 7. "حسابك محظور" ─────────────|
```

**طرق الحظر:**
1. **بالموبايل**: لو سجّل بموبايل محظور → ممنوع
2. **بالإيميل**: لو سجّل بإيميل محظور → ممنوع
3. **بالحساب**: عند تسجيل الدخول → نتحقق من `is_banned` ونطلعه

### هيكل الجلسة:

```
Supabase Auth (auth.users)
  └── id (UUID)
       └── users.id (نفس الـ ID)
            ├── anon_id: "مسافر #1234"
            ├── nickname: "اسم مستعار"
            ├── phone: "01012345678"      (للحظر)
            ├── phone_hash: SHA-256(...)   (للحماية)
            ├── email: "user@example.com" (للحظر)
            ├── is_banned: false           (للحظر)
            ├── user_type: "patient"
            └── ...
```

## 🚨 حل المشاكل

### مشكلة: "Email not confirmed"

**الحل**: تأكد إن **"Confirm email"** مفعّل في Supabase → Auth → Providers → Email

### مشكلة: ما وصلش كود على الإيميل

**الحل**: افحص:
1. مجلد **Spam** أو **Junk**
2. لو تستخدم Gmail → افتح **All Mail**
3. جرب تعيد الإرسال بعد دقيقة

### مشكلة: "Rate limit exceeded"

**الحل**: Supabase بتحدد عدد الـ OTP requests. انتظر ساعة.

### مشكلة: الـ RLS مش شغال

**الحل**: تأكد إنك شغّلت `supabase/schema.sql` — الكود بيعدّل كل الـ policies

### مشكلة: المستخدم المحظور قدر يسجل تاني

**الحل**: تأكد إن عمود `phone` في جدول `users` فيه الرقم الصحيح. ممكن تعدّله من Supabase Dashboard
